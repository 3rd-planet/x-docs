You are free to use any Node Package Manager available to you, however, we recommend using [npm](https://www.npmjs.com/) as it is the package manager we use internally. Some other popular package managers are [yarn](https://yarnpkg.com/) and [pnpm](https://pnpm.js.org/).

## Utilities

Framework X uses a number of utility packages to provide common functionality, these are listed below.

- [husky](https://www.npmjs.com/package/husky) - Git hooks made easy
- [nodemon](https://www.npmjs.com/package/nodemon) - Simple monitor script for use during development of a node.js app.
- [prettier](https://www.npmjs.com/package/prettier) - Prettier is an opinionated code formatter.
- [eslint](https://www.npmjs.com/package/eslint) - Find and fix problems in your JavaScript code.
- [dotenv](https://www.npmjs.com/package/dotenv) - Loads environment variables from .env file.


## Core packages

Framework X uses a number of core packages to provide common functionality, these are listed below.

- [@3rdplanet/x-core](https://github.com/3rd-planet/x-core) - Core package for Framework X
- [express](https://www.npmjs.com/package/express) - Fast, un-opinionated, minimalist web framework for node.
- [express-validator](https://www.npmjs.com/package/express-validator) - An express.js middleware for validator.
- [commander](https://www.npmjs.com/package/commander) - The complete solution for node.js command-line interfaces, inspired by Ruby's commander.
- [cors](https://www.npmjs.com/package/cors) - Node.js CORS middleware
- [express-async-errors](https://www.npmjs.com/package/express-async-errors) - A dead simple ES6 async/await support hack for ExpressJS
- [simple-node-logger](https://www.npmjs.com/package/simple-node-logger) - A simple lightweight logging library for Node.js
- [loading-cli](https://www.npmjs.com/package/loading-cli) - A simple loading spinner for cli programs.
